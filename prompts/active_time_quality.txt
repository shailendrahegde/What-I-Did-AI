# Active Time Quality Classification
# Classifies each user message into a mode showing how AI contributed.
# Used by the "How I Collaborated" section in the report.
#
# Two detection layers:
# 1. HAND-HOLDING: user correcting, redirecting, or AI producing errors
# 2. MODE: based on intent classification of the message content

[hand_holding_user_patterns]
# User expressing dissatisfaction or correcting AI output (regex, case-insensitive)
# "instead" removed (too broad — "instead of X use Y" is direction-setting, not error).
\b(no|wrong|not what i|that.s not|didn.t work|try again|still broken|not right|i said|i meant|i asked|that.s incorrect|you missed|go back|undo|revert|abort|stop|don.t|shouldn.t|not impressed|didn.t like|not what|why did you|the issue is|the problem is|that broke|you broke|can you fix|fix this|fix that|fix the|still not|i already told|read (my|the) (previous|last)|look at what i said)\b

[hand_holding_tool_patterns]
# Error signals in tool output (regex, case-insensitive)
\b(error|fail|exception|traceback|broken|crash|UnicodeError|SyntaxError|KeyError|TypeError|ModuleNotFound|ImportError|rejected|denied|exited with exit code [1-9])\b

[grunt_override_patterns]
# These fire BEFORE intent-based mode matching — always Grunt work regardless of other signals.
# Covers: documentation writes, readme updates, mass/bulk file operations.
\b(readme|documentation|changelog|docstring|write docs|update docs|write the docs|update the readme|write the readme|mass update|bulk update|rename all|update all|across all files|every file|all the files|all occurrences|find and replace|search and replace)\b

[modes]
# mode_name | intent_match | description
# Priority order: Hand-holding → Grunt override → Trivial → Intent-based (first match wins).
# Refinement FIRST over Building and Designing — iterating on existing output wins.
Refining     | Iterating                                  | Iterating, polishing, improving existing work
Designing    | Designing,Planning                         | Net-new design decisions, direction changes
Researching  | Researching,Investigating,Navigating       | Exploring options, investigating
Analyzing    | Analyzing                                  | Data analysis, image analysis, synthesizing results
Learning     | Learning                                   | Asking how-to, seeking explanations, getting recommendations
Reviewing    | Reviewing                                  | Code reviews, PR reviews, checking and validating work
Building     | Building                                   | Writing code, generating new features
Delegating   | Shipping,Configuring                       | Git ops, config, installs, docs, routine tasks
